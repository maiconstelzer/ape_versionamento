Comandos com o uso do FLOW

A tabela abaixo mostra os principais comandos do git (com FLOW).

Comando Descrição
git flow init Este comando faz a inicialização dentro de um repositório git já existente.
git flow feature start Funcionalidade Este comando cria uma nova branch com o nome "Funcionalidade" baseado na branch DEVELOP e faz o seu checkout automáticamente.
git flow feature finish Funcionalidade Este comando realiza o merge da branch "Funcionalidade" na branch DEVELOP, remove a branch "Funcionalidade" e faz checkout da branch DEVELOP.
git flow feature publish Funcionalidade Este comando realiza o push da branch "Funcionalidade" para o repositório remoto.
git flow feature pull Funcionalidade Este comando realiza o pull da branch "Funcionalidade".
git flow release start VER [HASH] Este comando cria uma branch da versão "VER" com base na branch DEVELOP, você pode opcionalmente fornecer um hash sha-1 do commit no lugar de "[HASH]".
git flow release publish VER Este comando realiza a publicação da versão "VER".
git flow release track VER Este comando é utilizado para verificar uma versão, no caso a versão "VER".
git flow release finish VER Este comando mescla a branch "VER" na master, em seguida cria uma tag com nome "VER", mescla a branch "VER" de volta na branch "DEVELOP" e remove a branch "VER".
git flow hotfix start HOTF [BRANCHBASE] Este comando marca "HOTF" como um novo hotfix, opcionalmente você pode especificar uma branch base em "[BRANCHBASE]".
git flow hotfix finish HOTF Este comando finaliza o hotfix "HOTF", mesclando tanto na branch DEVELOP como na MASTER. O merge realizado na master recebe uma TAG.

Observações

É sensato publicar o branch da versão depois de criá-lo, para permitir commits de outros desenvolvedores!

Você pode continuar a usar o git e todos os comandos dele normalmente como você já conhece, o git flow é apenas uma coleção de ferramentas!

A funcionalidade 'support' continua no estágio beta, seu uso não é indicado!
