Para que serve o commit e qual a melhor forma de utilizá-lo?

O commit é utilizado para confirmar suas alterações locais e adicionar uma mensagem que deve conter uma explicação do que foi alterado, quando gerado o commit, as confirmações estão apenas localmente, ou seja, no repositório nada foi alterado. A melhor forma de utilizar o commit segue a iniciativa de que a cada alteração relevante você faça uma confirmação, adicionando uma mensagem que explique com poucas palavras, tudo o que foi feito. Após todas as confirmações serem devidamente commitadas, é o momento de realização do “PUSH”, assim seu repositório vai receber todas as confirmações e as mensagens de explicação, mantendo assim, um grande histórico no projeto que poderá ser utilizado para verificar quando e porquê algo foi alterado e por quem está alteração foi feita.

Qual a diferença entre commit e push?

Commit e PUSH são comandos diferentes, a explicação mais rápida é a de que o COMMIT cria uma confirmação das alterações LOCAIS (OFFLINE) e permite adicionar uma mensagem explicativa sobre elas, já o PUSH é responsável por enviar essas alterações para o repositório remoto, permitindo que outros integrantes utilizem as novas modificações após o (FETCH + PULL). O commit grava confirmações localmente, o desenvolvedor pode realizar quantos commits achar necessário antes de realizar o PUSH para que as alterações fiquem gravadas no respositório, está forma de trabalho é muito útil, principalmente em casos onde você está fazendo uma determinada atividade e você vai precisar trocar de atividade e utilizar a MASTER sem alterações, neste caso, você cria uma BRANCH com suas alterações, realiza o COMMIT delas e depois faz o PUSH para que está branch fique em seu repositório. O PUSH tem como principal função, enviar suas confirmações locais para o repositório, afim de mantê-lo atualizado e que outros integrantes da equipe tenham acesso as suas alterações apenas com a sincronia (FETCH + PULL), porém, toda a magia está no COMMIT, pois o mesmo permite que o desenvolvedor escreva uma mensagen e fragmente suas alterações, mantendo um histórico de quando, porquê e por quem foram feitas.
